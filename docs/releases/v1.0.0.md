# Lunie v1.0.0

## Overview

Lunie v1.0.0 is the first public release of the project.

Lunie is a self-hosted workflow automation engine for technical users who want local control, an API-first workflow model, and inspectable execution history. This release brings together the API server, worker, CLI, and terminal UI into a release-ready baseline that can be installed, bootstrapped, and used end to end.

## Highlights

- First public launch of Lunie
- Self-hosted workflow execution with server, worker, CLI, and TUI
- End-to-end workflow authoring and execution with `http`, `transform`, and `condition` steps
- Local stack bootstrap with `lunie init`, including optional bundled Postgres and Redis
- Packaged distribution flow for GitHub releases, Docker images, Homebrew, AUR, and Cloudsmith

## Install

### Homebrew

```bash
brew tap gentij/lunie
brew install lunie
```

### AUR

```bash
yay -S lunie-cli-bin
```

### GitHub Release Binary

Download the matching archive from:

- https://github.com/gentij/lunie/releases

Extract `lunie` and put it on your `PATH`.

## Upgrade Notes

- This is the first public Lunie release, so there is no public migration path to follow.
- Operators can bootstrap a local stack with `lunie init --with-local-datastores` or point Lunie at existing Postgres and Redis instances.
